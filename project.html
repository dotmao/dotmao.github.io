NOVA

A conceptual cloud storage service web application that focuses on improving file organization and collaboration controls.

Concept Overview:

Digital content has increased exponentially over the recent years, people are constantly seeking better ways content storage and creation online and more and more teams are in need of smooth online workflow and proper digital content management system. Cloud storage services become one solution to the problem. However, many frustrations such as poor file organization and lack of collaboration control still remain in the market.

Challenges:

“I have millions of files in the cloud. It takes me forever to find the file I want.” –Jason


Difficult to locate files.

“Sometimes, I get frustrated during work when I need to look for certain information but the teammates are updating the file in real-time that makes it difficult for me because the file keeps changing constantly.” –Felix


Poor collaboration controls.

Solutions:

Nova is designed to close some of the gaps mentioned above and offer users an intuitive interface to store, share, access and create content on one collective platform. It provide smooth content management system by establishing file categories and adding tag feature at the same time, Nova ensures collaboration accuracy by granting file monitoring and auditing abilities to the owner of the file such as project manager.

Discovery . User Research

In order to validate my initial assumptions, I needed to get the answers from my users then understand what they need and how they use current services. I conducted an online user survey and distributed on social media in order to optimize my response rate. The survey was designed with the following objectives in mind:

1.) Who are the main competitors out there?

2.) Why do users like using cloud storage services?

3.)If they are not using cloud storage, what would encourage them to use it?

4.) What are some concerns they have over cloud storage services?

5.) What can cloud storage services do better for users?

It was very obvious that cloud storage service needs to improve security features at the same time provides more a user-friendly interface for collaboration and file organization.

Discovery . Competitors

In order to properly position this new service, it is vital to conduct a competitive analysis. I used SWOT analysis method for this project to carefully compare competitors’ strengths and weaknesses, at the same time to seek opportunities in the industry. Check out my competitive analysis here.

Discovery . User Persona

Before digging deeper into what users need, I needed to find out who our users are and create user personas. I interviewed different sets of users and concluded that there are 3 different but somewhat similar personas:

  • 1.) Tech-savvy individuals who use cloud storage to store and organize all files in one place.
  • 2.) Young professionals who use cloud storage to collaborate with colleagues for work.
  • 3.) Entrepreneurs or start ups who rely on cloud storage for smooth workflow.

Information Architecture . User Stories & User Flow

In order to better visualize the infrastructure of the site, I crafted user flows started from simple sketch to refined digital format. Then I created a list of user stories to prioritize user tasks and build MVP(minimum viable product). I mapped out the easiest and the most common path for users to sign up and log in to their account, how to upload from the dashboard, create content from the dashboard, organize content and managing account settings.

Information Architecture . Sketch

Design does not happen on the screen first. Almost all projects start with some quick sketches on paper or whiteboards. It is basic, fast. and easy to complete and it allows ideas flow freely with just a pen in hand. There is no distraction from design details such as colors, fonts and spacing in sketching. The procedure solely focuses on idea generation.

The low-fidelity mockup is quick and rough, it allowed me to generate some quick ideas and build a simple blueprint of the site without any established visual assets. I started with some quick sketches, then I completed a refined version wireframes in myBalsamiq.

Visual Design . Branding

Although it is a concept, a brand identity is still necessary. I started mapping out name associations, soon enough, I focused on the idea of “space” due to its direct link to cloud service. I chose the brand name “NOVA”, which represents brightness, brilliance and infinity. The brand identity was born accordingly. NOVA, the new star in cloud service industry, conveys advanced technology, provides seamless workflow and is committed to being transparent with all users.

With the brand values and our target market in mind, I decided to choose blue as our main color scheme. However, the traditional vibrant blue that is often used for large corporations was not the best fit for the audience. I wanted a color that speaks trust and innovation. After a few trials, I found that the combination of Tiffany blue with light green shade was a very interesting color for this brand. It is very modern, stylish and a subtle color for screen use. I designed the logo in different versions based on a cloud shape I drew and ran a preference test. I then established my style guide for the site according to the result of my logo preference test.

Visual Design . High Fidelity Wireframes

With all my visual assets completed, I created a series of high-fidelity mockups for both large screen and mobile screen.

The preview of text-type files such as word document, excel or PDF is not particularly helpful due to lack of legibility of the content. In order to reduce clutter, I decided to only show preview for visual-file such as images and videos.
The dashboard emphasizes file organization and collaboration controls. The left panel allows users to view files in different categories, this feature can reduce search time by filtering file types.

Reflection:

As my first complete UX and UI design project, it has been an incredible learning experience. I gained hands-on insights on the importance of each design stage and how they tie with each other as a whole to make seamless user experience. I particularly enjoyed how I was able to use the data collected from user survey and interview to make some crucial design decisions. As a former marketer, it was natural to design this project with business goals in mind, which is a great practice when crafting brand identity and analyze target market.


Further improvements: With limited time and resources, I did not get the amount of survey response I wanted. Therefore, if time permits I would conduct more a thorough user survey to distill user insights by asking more specific questions and distribute the survey in more channels. For qualitative research, I would also like to interview more users remotely and in-person to study their behavior while using competitor’s service.
